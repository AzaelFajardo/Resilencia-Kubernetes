## Descripción general
Sistema de microservicios que simula una plataforma de pedidos en línea,
desplegado localmente con Docker Compose y posteriormente en Kubernetes.

## Microservicios
| Servicio             | Puerto local | Responsabilidad                        |
|----------------------|-------------|----------------------------------------|
| order-service        | 8000        | Orquestador principal del flujo        |
| user-service         | 8001        | Validación y consulta de usuarios      |
| inventory-service    | 8002        | Control de stock y reservas            |
| payment-service      | 8003        | Procesamiento simulado de pagos        |
| notification-service | 8004        | Envío de confirmaciones                |

## Herramientas de observabilidad
| Herramienta  | URL local                  | Propósito                        |
|--------------|---------------------------|----------------------------------|
| Prometheus   | http://localhost:9090      | Recolección de métricas          |
| Grafana      | http://localhost:3000      | Visualización (admin/admin)      |
| Jaeger UI    | http://localhost:16686     | Trazas distribuidas              |

## Variables de caos por servicio
Cada servicio acepta estas env vars para simular fallas controladas:
- `FAILURE_RATE` — probabilidad de fallo aleatorio (0.0 a 1.0)
- `LATENCY_MS`   — latencia artificial en milisegundos
- `TIMEOUT_RATE` — probabilidad de timeout (solo payment-service)
