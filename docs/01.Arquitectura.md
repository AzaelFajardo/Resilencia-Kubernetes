## General Description
Microservices system that simulates an online ordering platform,
deployed locally with Docker Compose and later on Kubernetes.

## Microservices
| Service | Local Port | Responsibility |
|---------|------------|----------------|
| order-service | 8000 | Main flow orchestrator |
| user-service | 8001 | User validation and lookup |
| inventory-service | 8002 | Stock control and reservations |
| payment-service | 8003 | Simulated payment processing |
| notification-service | 8004 | Confirmation delivery |

## Observability Tools
| Tool | Local URL | Purpose |
|------|-----------|---------|
| Prometheus | http://localhost:9090 | Metrics collection |
| Grafana | http://localhost:3000 | Visualization (admin/admin) |
| Jaeger UI | http://localhost:16686 | Distributed tracing |

## Chaos Variables by Service
Each service accepts these environment variables to simulate controlled failures:
- `FAILURE_RATE` - probability of random failure (0.0 to 1.0)
- `LATENCY_MS` - artificial latency in milliseconds
- `TIMEOUT_RATE` - probability of timeout (payment-service only)
