# Microservice: Notification Service

## 1. Description

The Notification Service is a microservice responsible for simulating notification delivery within the system. Its main role is to confirm events generated by other services, such as order creation.

## 2. Objective

Provide a simple mechanism for sending notifications, allowing the communication flow between microservices to be completed.

## 3. Functionality

This service exposes an HTTP endpoint that allows:

- Sending a notification
- Returning a response indicating that the notification was delivered

## 4. Available Endpoint

`GET /notify`

Description:

Simulates the delivery of a notification.

Response:

```json
{
  "status": "sent",
  "message": "Notification delivered"
}
```

## 5. Technologies Used

- Python
- FastAPI
- Uvicorn

## 6. Execution

Local:

```bash
python -m uvicorn main:app --reload --port 8001
```

Docker:

The service runs inside a container and exposes port 8000 internally.

## 7. Role in the Architecture

This microservice is used by the Order Service to notify the user when an order has been processed.

## 8. Considerations

- It does not handle complex logic
- It does not present simulated failures
- Its purpose is to represent a simple external service
